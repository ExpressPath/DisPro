Dispro accepts files, folders, URLs, code, and proof artifacts, then estimates cost from size and workload profile.

API

Order processing from apps, CLIs, build systems, and external services.

POST /orders
{
  "customerId": "customer_001",
  "workload": "custom.thumbnail.extract",
  "source": {
    "kind": "url",
    "uri": "https://example.com/input.bin",
    "byteSize": 5500000,
    "contentHash": "d6f8d5c842ec2c59..."
  },
  "verificationLevel": "standard",
  "requirements": {
    "maxChunkBytes": 2000000,
    "workloadProfile": {
      "unitBytes": 1000000,
      "estimatedMemoryGb": 1
    }
  }
}

The API creates an order plan immediately and persists tasks, assignments, and audit events.

  • GET /orders/:id/tasks returns worker-ready tasks.
  • GET /orders/:id/audit verifies the audit hash chain.
  • POST /nodes/register registers generic or specialized nodes.

Network

Phones verify, PCs compute, and trusted servers handle high-value work.

Light verification

Signatures, hashes, random checks, and canary tasks are routed to low-load verification nodes.

General compute

Image conversion, PDF compression, data transforms, code tests, and custom workloads are assigned by score.

Trusted execution

Enterprise, urgent, long-running, and GPU-heavy jobs can be restricted to high-trust node pools.
