Platform

注文から結果までを、処理単位で設計する。

01

Order

ファイル、フォルダ、URL、コード、証明ファイルを注文として受け取り、容量と処理内容から価格を計算します。

02

Split

任意のワークロードをチャンクへ分け、compute、verification、canary タスクとして配布できる計画に変換します。

03

Verify

ランダム照合、既知答えタスク、署名ログで、毎回全件再計算せずに正しさを追跡します。

API

サービスや開発環境から、そのまま処理を注文。

API は注文計画を即時生成し、タスク、割り当て、監査イベントを保存します。

  • GET /orders/:id/tasks で Worker 向けタスクを取得
  • GET /orders/:id/audit で監査チェーンを検証
  • POST /nodes/register で汎用ノードを登録
